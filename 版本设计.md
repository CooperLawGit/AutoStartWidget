# AutoStartWidget 版本设计

## 产品定位

AutoStartWidget 是开机自启的办公小组件集合管理器。集合管理器只负责统一入口、模块开关、配置保存和生命周期管理。

截图工具和护眼助手是平级模块。每个模块独立开关、独立配置、独立调用。模块关闭时不加载运行时资源，不注册热键，不启动计时器，不启动外部进程。

## 模块原则

- 集合管理器常驻托盘，提供统一菜单。
- 截图工具开启后才加载截图能力，关闭后释放截图窗口、热键和截图资源。
- 护眼助手开启后才加载计时器和锁屏事件监听，关闭后释放相关资源。
- 开机自启只启动 AutoStartWidget 集合管理器。各模块是否启动由模块开关配置决定。
- 截图工具是 .NET 8 WinForms 原生 C# 模块，参考 SETUNA2 行为，不再依赖外部 `SETUNA.exe`。
- 图标统一使用 `D:\WorkSpace\GithubProject\icon.png` 生成并嵌入应用资源。

## 当前状态

当前版本已具备 AutoStartWidget 托盘入口、护眼助手基础能力、原生截图核心能力和 SETUNA2 legacy 源码参考。

截图能力已迁入 AutoStartWidget 进程内。SETUNA2 legacy 源码保留为后续扩展迁移参考。

## 版本节奏

### v0.1 集合管理器基础

- 建立 AutoStartWidget 项目结构。
- 提供托盘入口、开机自启、本地配置。
- 接入护眼助手基础功能。
- 保留 SETUNA2 legacy 源码，作为截图重写参考。

### v0.2 原生截图核心能力

- 使用 .NET 8 WinForms 重写截图工具核心流程。
- 支持区域截图、Esc 取消、截图贴片、贴片拖动、关闭、复制、另存。
- 支持截图模块独立启用/禁用。
- 支持截图热键注册/注销。
- 支持窗口识别/窗口截图辅助。
- 支持废纸篓：关闭截图后可恢复、清空。
- 不再把外部 `SETUNA.exe` 作为目标架构。

### v0.3 护眼助手模块化

- 把护眼助手改为平级模块。
- 护眼模块关闭时不创建计时器，不订阅系统锁屏事件。
- 护眼设置归入集合管理器统一配置。
- 托盘菜单提供护眼模块开关和立即护眼入口。

### v0.4+ SETUNA2 扩展能力补齐

- 按使用价值逐步迁移 SETUNA2 非核心扩展。
- 优先补齐截图贴片体验和常用格式能力。
- 低频、历史服务类能力放到长期计划。

## 发布约束

- 本地开发先保证主应用可构建。
- README 必须区分已实现能力和后续扩展能力。
- 不把 legacy SETUNA2 外部进程描述成运行架构。
